# Java高级学习计划

这是一份面向中级开发者的 Java 高级学习计划，目标不是“看完很多知识点”，而是建立一套完整的高级 Java 能力体系，包括语言特性、并发编程、JVM、框架原理、数据库与中间件、系统设计、性能调优和工程实践。

## 1. 学习目标

完成这份计划后，应该具备以下能力：

- 能清楚解释 Java 核心机制，如泛型、反射、注解、代理、集合源码
- 能从原理层面理解并发模型、线程池、锁、AQS、JMM
- 能独立分析 JVM 内存模型、类加载、GC、性能问题
- 能熟练使用 Spring / Spring Boot，并理解 IoC、AOP、事务、自动装配原理
- 能完成一个具备数据库、缓存、消息队列、监控能力的后端项目
- 能在面试和实际工作中回答“为什么这样设计”，而不是只会调用 API

## 2. 适用人群

适合已经掌握以下基础的开发者：

- 熟悉 Java 基础语法、面向对象、异常、集合
- 能使用 Maven 或 Gradle
- 写过基础的 Web 项目或后端接口

如果你目前还停留在“只会 CRUD”的阶段，这份计划很适合拿来做进阶路线图。

## 3. 学习原则

高级学习阶段，不建议只刷文章或只看视频，建议采用下面这套方式：

- 30% 学原理：看书、官方文档、源码、经典文章
- 40% 写代码：最小 demo、手写实现、实验代码
- 20% 做项目：把技术点串起来
- 10% 做总结：写笔记、画图、复盘问题

学习时坚持四个动作：

- 学完一个概念，必须自己复述一遍
- 学完一个模块，必须写一个最小 demo
- 学完一个阶段，必须产出一篇总结
- 学完一个大主题，必须能回答常见面试题

## 4. 总体安排

建议周期为 24 周，大约 6 个月。  
如果你平时工作较忙，可以拉长到 32 周；如果你每天都有稳定时间，也可以压缩到 16 周。

建议每周投入：

- 工作日每天 1.5 到 2 小时
- 周末至少 4 到 6 小时

每周总时长最好在 12 到 16 小时之间。

## 5. 分阶段学习路线

## 第一阶段：Java语言与基础能力深化（第 1-4 周）

目标：从“会用 Java”升级到“理解 Java 设计与实现方式”。

重点内容：

- 泛型、类型擦除、边界限定
- 集合体系与常见源码：ArrayList、LinkedList、HashMap、ConcurrentHashMap
- equals、hashCode、compareTo 的设计原则
- 异常体系、受检异常与非受检异常
- 反射、注解、动态代理
- I/O 与 NIO 基础

阶段产出：

- 手写一份集合源码核心笔记
- 写一个基于动态代理的 demo
- 总结一篇“Java基础高级知识梳理”

## 第二阶段：并发编程与多线程核心（第 5-8 周）

目标：掌握 Java 并发编程的底层思想，而不是只记住几个 API。

重点内容：

- 线程创建方式与线程状态
- synchronized、volatile、CAS
- Java 内存模型 JMM
- ReentrantLock、ReadWriteLock、StampedLock
- AQS 原理
- 线程池参数设计与拒绝策略
- ConcurrentHashMap、CountDownLatch、CyclicBarrier、Semaphore
- CompletableFuture

阶段产出：

- 手写一个简化版线程池
- 画出 JMM、AQS、线程池执行流程图
- 总结一篇“Java并发常见问题与解决方案”

## 第三阶段：JVM 与性能调优（第 9-12 周）

目标：建立对 JVM 的完整认知，学会定位内存、GC、类加载问题。

重点内容：

- JVM 运行时数据区
- 对象创建过程、对象头、内存分配
- 类加载机制、双亲委派模型
- 字节码基础与常见指令
- 垃圾回收算法与常见收集器
- G1、ZGC 的基本特点
- JVM 常用参数
- 性能分析工具：jstack、jmap、jstat、jcmd、VisualVM、Arthas

阶段产出：

- 写一篇“JVM内存模型与GC机制”总结
- 做一次本地 OOM / 死锁 / CPU 飙高实验
- 输出一份性能分析报告

## 第四阶段：数据库与中间件能力（第 13-16 周）

目标：补齐高级后端工程师必备的数据层能力。

重点内容：

- JDBC、连接池原理
- MySQL 索引、事务、锁、隔离级别
- SQL 优化思路
- Redis 数据结构、缓存设计、缓存一致性
- 消息队列基础：RabbitMQ 或 Kafka
- 分布式场景下的幂等、重试、补偿

阶段产出：

- 做一个数据库性能优化案例
- 做一个 Redis 缓存实战 demo
- 做一个消息队列异步解耦 demo

## 第五阶段：Spring 体系与框架原理（第 17-20 周）

目标：不仅会用 Spring Boot，还要知道它为什么这样工作。

重点内容：

- Spring IoC 容器
- Bean 生命周期
- AOP 原理
- 事务传播行为与事务失效场景
- Spring Boot 自动配置原理
- Starter 机制
- Spring MVC 请求处理流程
- 参数校验、统一异常处理、拦截器、过滤器
- Spring Security 基础

阶段产出：

- 画出 Spring Bean 生命周期图
- 手写一个简化版 IoC / AOP demo
- 总结一篇“Spring Boot 自动装配原理”

## 第六阶段：架构设计与工程实践（第 21-24 周）

目标：把知识点串成完整工程能力。

重点内容：

- 设计模式在 Java 中的落地
- 分层架构、领域建模、模块拆分
- 接口设计与错误码设计
- 日志、链路追踪、监控、告警
- 单元测试、集成测试
- Docker 基础部署
- 限流、熔断、降级
- 高可用、高并发、高扩展的基本思维

阶段产出：

- 完成一个完整的后端项目
- 项目需要包含：认证、数据库、缓存、异步任务、日志、异常处理、测试
- 输出一份项目复盘文档

## 6. 推荐实战项目

为了真正进入“高级”阶段，至少要做 1 到 2 个完整项目。推荐方向：

- 权限管理系统：用户、角色、菜单、鉴权、日志审计
- 电商订单系统：库存、订单、支付状态流转、消息队列
- 博客或知识库系统：全文检索、缓存、评论、后台管理
- 秒杀系统练习版：缓存、限流、异步削峰、热点数据处理

项目要求：

- 使用 Spring Boot
- 接 MySQL
- 接 Redis
- 至少引入一种消息机制
- 具备基础测试、日志和异常治理

## 7. 每周学习节奏建议

可以按照下面的节奏推进：

- 周一：学习理论与概念
- 周二：阅读源码或官方文档
- 周三：写 demo 验证结论
- 周四：整理笔记与问题
- 周五：做阶段复盘
- 周末：做实战项目或专题练习

这样安排的好处是不会陷入“只学不练”，也不会陷入“只写代码不理解原理”。

## 9. 阶段自检标准

学到后面最怕“看起来学了很多，实际上不会用”。  
每完成一个阶段，至少要检查下面这些问题：

- 我能不能不看资料，把核心原理讲清楚
- 我能不能写一个最小 demo 证明自己真的会
- 我能不能把这个知识点用到项目里
- 我能不能说出常见问题、风险点和优化方式

如果四个问题里有两个答不上来，就说明这一阶段还没真正掌握。

## 10. 最终目标画像

当你完成这份计划后，理想状态下应该达到下面这个水平：

- 能独立负责中大型 Java 后端模块
- 能分析线上常见性能与并发问题
- 能解释底层原理，而不是停留在使用层
- 能把框架、中间件、数据库和系统设计串起来
- 能胜任高级 Java 开发岗位的大部分技术要求

## 11. 一句话建议

Java 的高级，不是记住更多类库，而是逐步建立三种能力：

- 原理理解能力
- 工程落地能力
- 问题分析能力

如果你愿意长期投入，这份计划完全可以作为未来半年到一年的主线学习路线。
