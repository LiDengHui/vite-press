# 线程模型与性能优化

## 1. 这是什么

Netty 的线程模型围绕 EventLoop 组织，性能优化则围绕线程切换、内存分配和 I/O 处理效率展开。  
这是 Netty 高性能的核心来源之一。

## 2. 为什么重要

如果线程模型用错，再好的非阻塞框架也可能跑不快。  
理解性能优化思路，才能避免把 Netty 用成普通阻塞程序。

## 3. 核心内容

- Boss 与 Worker 线程角色
- EventLoop 绑定关系
- 任务提交与异步执行
- 零拷贝基础认知
- 内存池与对象复用

## 4. 学习重点

- 理解 I/O 线程不要承载重业务
- 理解线程模型和吞吐直接相关
- 理解性能优化很多来自减少复制与切换

## 5. 常见问题

- 在 I/O 线程中执行耗时任务
- 忽视内存池和直接内存的作用
- 把所有性能问题都归因于框架本身

## 6. 练习建议

- 分析一次耗时逻辑阻塞 EventLoop 的后果
- 总结 Netty 常见性能优化点
- 对比有无内存池的差异认知

## 7. 自测问题

- 为什么 I/O 线程不适合执行重业务
- Boss 和 Worker 通常各自承担什么职责
- Netty 性能优化通常从哪些点切入
