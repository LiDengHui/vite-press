# 架构与事件驱动

## 1. 这是什么

Netty 的核心是事件驱动和 Reactor 模型。  
它通过事件循环和非阻塞 I/O，把大量连接的处理统一组织起来。

## 2. 为什么重要

如果不理解事件驱动，就很难真正理解 Netty 为什么比传统阻塞模型更适合高并发网络服务。  
这也是后续理解 EventLoop 和 Channel 的前提。

## 3. 核心内容

- Reactor 思想
- 事件驱动
- EventLoopGroup
- 非阻塞 I/O 基础认知
- Netty 整体架构

## 4. 学习重点

- 理解 Netty 的高效来自模型设计而非单个 API
- 理解事件循环如何减少线程切换
- 理解为什么一个 EventLoop 可以管理多个连接

## 5. 常见问题

- 只记组件名，不理解执行模型
- 把 Netty 仅理解为 socket 封装
- 不清楚事件驱动和阻塞模型的根本差异

## 6. 练习建议

- 画一张 Netty 事件驱动架构图
- 对比阻塞模型和事件驱动模型
- 总结 EventLoop 在整体中的职责

## 7. 自测问题

- Netty 为什么适合高并发连接场景
- EventLoop 为什么是 Netty 的核心
- Reactor 模型解决了什么问题
