# 粘包拆包与协议设计

## 1. 这是什么

TCP 是字节流协议，不保证消息边界，因此会出现粘包和拆包现象。  
协议设计和解码策略就是为了解决这个问题。

## 2. 为什么重要

很多网络程序“偶发解析错误”的根源都在这里。  
如果协议边界不清楚，再好的业务逻辑也无法稳定运行。

## 3. 核心内容

- TCP 字节流特性
- 定长协议
- 分隔符协议
- 长度字段协议
- 半包与拆包处理

## 4. 学习重点

- 理解粘包拆包不是 Netty bug，而是 TCP 特性
- 理解协议设计应优先保证边界清晰
- 理解长度字段协议在通用业务里的适用性

## 5. 常见问题

- 误以为一次 read 就是一条完整消息
- 协议设计没有明确边界字段
- 业务代码直接处理半包数据

## 6. 练习建议

- 设计一个带长度字段的简单协议
- 用 Netty 的解码器处理半包场景
- 总结几种常见协议边界方案

## 7. 自测问题

- 粘包拆包为什么是 TCP 天然问题
- 为什么长度字段协议常被广泛采用
- 协议设计为什么决定后续处理复杂度
