# ByteBuf与编解码

## 1. 这是什么

ByteBuf 是 Netty 对字节缓冲区的增强抽象，编解码则负责把网络字节流转换成业务对象。  
这是协议处理和性能优化的重要基础。

## 2. 为什么重要

网络通信最终都落在字节流上。  
如果不了解 ByteBuf 和编解码，就很难设计稳定的协议处理链。

## 3. 核心内容

- ByteBuf 的读写指针
- 堆内存与直接内存
- 引用计数
- 编码器与解码器
- 序列化基础认知

## 4. 学习重点

- 理解 ByteBuf 相比 ByteBuffer 的改进点
- 理解引用计数和内存释放的重要性
- 理解编解码在协议边界上的作用

## 5. 常见问题

- 忽视引用计数导致内存问题
- 编解码逻辑和业务逻辑混在一起
- 不理解直接内存对性能的意义

## 6. 练习建议

- 写一个简单字符串协议编码器和解码器
- 观察 ByteBuf 读写指针变化
- 总结堆内存和直接内存的差异

## 7. 自测问题

- ByteBuf 为什么更适合网络编程
- 引用计数为什么是高频风险点
- 编解码为什么应该尽量和业务处理分层
