# 治理能力与配置中心

## 1. 这是什么

治理能力包括路由、限流、降级、动态配置等，配置中心用于统一管理服务运行配置。  
这是 Dubbo 从“能调用”走向“能运营”的关键部分。

## 2. 为什么重要

服务数量一多，光靠代码配置很快就会失控。  
治理能力让服务体系具备动态调整和统一管控的能力。

## 3. 核心内容

- 动态配置
- 服务路由
- 限流与降级
- 黑白名单基础认知
- 配置中心作用

## 4. 学习重点

- 理解治理能力服务的是稳定性和运维效率
- 理解配置中心是统一控制面的一部分
- 理解动态治理意味着线上行为可以不发版就调整

## 5. 常见问题

- 配置散落在多个服务中
- 没有统一治理入口
- 治理规则修改缺乏审计和验证

## 6. 练习建议

- 设计一套服务限流与降级思路
- 总结配置中心在服务治理中的价值
- 画一张 Dubbo 治理能力示意图

- 为什么服务系统需要治理能力而不只是调用能力
- 配置中心在 Dubbo 体系中解决了什么问题
