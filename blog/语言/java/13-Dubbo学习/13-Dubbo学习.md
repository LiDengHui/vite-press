# Dubbo学习

Dubbo 是 Java 生态里非常有代表性的 RPC 框架。  
学习 Dubbo 的重点，不只是把服务调起来，而是理解服务暴露、注册发现、负载均衡、容错和治理体系。

## 1. 学习定位

这一组内容用于补充 Java 服务化与 RPC 方向的能力。  
目标是让你理解 Dubbo 怎样把本地调用抽象成远程调用，以及它在服务治理上的价值。

## 2. 学习重点

- 理解 Dubbo 的调用链路和角色分工
- 理解注册中心在服务体系里的作用
- 理解负载均衡和容错如何影响可用性
- 理解协议和序列化为什么影响性能
- 理解 Dubbo 的治理能力为什么重要

## 3. 建议顺序

1. 基础架构与调用链路
2. 服务暴露与服务发现
3. 负载均衡与容错机制
4. 序列化与通信协议
5. 治理能力与配置中心
6. 性能调优与排障

## 4. 阶段产出

- 一份 Dubbo 调用链路图
- 一份服务治理能力清单
- 一份 RPC 性能与排障笔记

## 5. 自测标准

- 能解释 Dubbo 为什么是 RPC 框架而不是简单 HTTP 调用
- 能说清注册中心、负载均衡、容错各自做什么
- 能说明远程调用为什么必须重视治理和排障
