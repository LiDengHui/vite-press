# 基础架构与调用链路

## 1. 这是什么

Dubbo 基础架构描述了服务提供者、服务消费者、注册中心和监控治理组件如何协同。  
调用链路则描述一次 RPC 调用从发起到返回的大致过程。

## 2. 为什么重要

只有把整体链路看清楚，后面的注册发现、容错、协议和治理才能串起来。  
这是理解 Dubbo 的第一张总图。

## 3. 核心内容

- Provider
- Consumer
- Registry
- Monitor 基础认知
- 调用代理与网络通信链路

## 4. 学习重点

- 理解 Dubbo 是把远程调用包装成本地接口调用体验
- 理解注册中心不是调用中转，而是地址发现中心
- 理解调用链路是代理、路由、网络、序列化共同完成的

## 5. 常见问题

- 把 Dubbo 理解成单纯的 HTTP 封装
- 不清楚注册中心在什么时候起作用
- 忽视代理层在调用体验中的作用

## 6. 练习建议

- 画一张 Dubbo 基础架构图
- 描述一次 Consumer 调用 Provider 的完整路径
- 总结 Dubbo 与普通 HTTP 调用的差异

## 7. 自测问题

- Dubbo 为什么能让远程调用看起来像本地调用
- 注册中心和 Provider 分别承担什么职责
- 一次 RPC 调用通常会经过哪些关键环节
