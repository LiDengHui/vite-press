# 性能调优与排障

## 1. 这是什么

Dubbo 性能调优与排障关注的是调用延迟、超时、错误率和资源消耗等问题的分析与治理。  
这是服务化系统落到生产环境后的必备能力。

## 2. 为什么重要

RPC 问题具有链路长、影响广、定位难的特点。  
如果没有系统化排障思路，很容易只看到表面超时，看不到真正瓶颈。

## 3. 核心内容

- 超时问题
- 重试放大问题
- 序列化与网络延迟
- 线程池与请求积压
- 监控指标与链路分析

## 4. 学习重点

- 理解 Dubbo 问题常常是链路问题不是单点问题
- 理解超时、重试、线程池要一起分析
- 理解排障需要监控、日志和调用链协同

## 5. 常见问题

- 请求慢就一味增大超时
- 重试把下游故障放大
- 缺少关键链路监控

## 6. 练习建议

- 设计一份 Dubbo 排障清单
- 复盘一次典型 RPC 超时场景
- 总结 Dubbo 常见性能问题的定位路径

## 7. 自测问题

- 为什么 RPC 超时常常不是单点问题
- 重试为什么有时会让系统更糟
- Dubbo 排障时最值得看的指标有哪些
