# 核心概念与倒排索引

## 1. 这是什么

ElasticSearch 的核心是倒排索引模型。  
它不是按行扫描数据，而是先把词项和文档之间的关系建立起来，再反向查找匹配结果。

## 2. 为什么重要

如果不理解倒排索引，就很难真正理解为什么搜索引擎和数据库查询思路不同。  
这也是理解全文检索性能来源的起点。

## 3. 核心内容

- Index、Document、Field
- Term 与 Token
- Analyzer 的基本作用
- 倒排索引结构
- 搜索相关性基础认知

## 4. 学习重点

- 理解 ElasticSearch 的“索引”概念和数据库索引不同
- 理解分词为什么会影响搜索结果
- 理解倒排索引为何适合关键词检索

## 5. 常见问题

- 把 ES 索引简单等同于 MySQL 索引
- 不理解分词器对查询结果的影响
- 只会用 API，不理解底层模型

## 6. 练习建议

- 画一张倒排索引结构图
- 选一段文本手工拆解词项和文档关系
- 比较数据库精确查找和倒排搜索的差异

## 7. 自测问题

- 倒排索引为什么适合全文搜索
- 分词器在 ES 中承担什么职责
- ES 的 Index 和数据库索引有什么本质差异
