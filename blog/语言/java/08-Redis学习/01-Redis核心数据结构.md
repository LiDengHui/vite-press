# Redis核心数据结构

## 1. 这是什么

Redis 提供了多种内存数据结构来支撑不同业务场景。  
数据结构选型，是 Redis 使用质量的起点。

## 2. 为什么重要

同样一份业务数据，用不同结构表示，性能、空间和操作复杂度都会不同。  
选错结构，后面的扩展和优化都会变难。

## 3. 核心内容

- String
- Hash
- List
- Set
- ZSet
- Bitmap 与 HyperLogLog 的基本认知

## 4. 学习重点

- 把结构能力和业务模型对应起来
- 理解不同结构的时间复杂度和应用边界
- 不把 Redis 简化成“字符串缓存”

## 5. 常见问题

- 无论什么场景都用 String
- 不理解 ZSet 在排序场景的优势
- 忽视结构带来的空间开销

## 6. 练习建议

- 用不同结构实现排行榜、去重、计数
- 总结每种结构的适用场景
- 对比一个场景下多种结构的利弊

## 7. 自测问题

- ZSet 为什么适合排行榜
- Hash 和 String 在对象存储上各有什么特点
- 什么业务更适合 Set 或 Bitmap
