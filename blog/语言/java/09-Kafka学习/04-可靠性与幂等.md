# 可靠性与幂等

## 1. 这是什么

可靠性与幂等关注的是消息是否会丢、是否会重、业务是否能安全重复处理。  
这是 Kafka 在生产系统里最关键的能力之一。

## 2. 为什么重要

消息系统最怕的不是“偶尔慢一点”，而是“消息丢了”或“业务重复执行”。  
可靠性和幂等，决定了 Kafka 能否承载关键链路。

## 3. 核心内容

- `acks`
- 重试
- 幂等生产者
- 事务消息基础认知
- 至少一次、至多一次、恰好一次语义认知
- 消费端幂等

## 4. 学习重点

- 理解可靠性和吞吐通常存在权衡
- 理解“恰好一次”不是简单一个开关
- 理解消费端幂等常常比中间件配置更关键

## 5. 常见问题

- 只调 `acks` 不看整体链路
- 误以为开启幂等后业务就绝对不会重复
- 忽视消费端去重与状态控制

## 6. 练习建议

- 总结 Kafka 常见消息语义的适用场景
- 设计一个消费端幂等方案
- 对比生产端配置对可靠性的影响

## 7. 自测问题

- `acks` 配置会如何影响可靠性和性能
- 为什么消息系统里幂等几乎是必备能力
- “恰好一次”为什么比想象中复杂
