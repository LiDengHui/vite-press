# 分区副本与顺序性

## 1. 这是什么

分区决定并发和扩展能力，副本决定容灾和可靠性，顺序性则是业务经常关心的消费属性。  
这三者是 Kafka 设计权衡的核心。

## 2. 为什么重要

一旦业务同时要求高吞吐、高可靠和严格顺序，就必须理解这三者之间的关系。  
否则系统设计很容易出现错误预期。

## 3. 核心内容

- 分区扩展能力
- 副本机制
- ISR 基本认知
- 分区内顺序
- 多分区下的顺序边界

## 4. 学习重点

- 理解顺序性通常只能在单分区范围内强保证
- 理解副本机制会影响写入确认与可用性
- 理解分区越多并不意味着所有问题都更好

## 5. 常见问题

- 既想全局严格顺序又想无限扩展
- 不理解副本同步延迟带来的影响
- 分区键设计不稳定导致顺序失真

## 6. 练习建议

- 为一个订单场景设计分区键
- 说明为什么单分区更容易保证顺序
- 总结分区与顺序性之间的权衡

## 7. 自测问题

- Kafka 的顺序性通常能保证到什么粒度
- 分区和副本分别更偏向解决什么问题
- 为什么业务分区键设计很关键
