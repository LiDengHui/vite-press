# Kafka性能调优

## 1. 这是什么

Kafka 性能调优关注吞吐、延迟、稳定性和资源利用之间的平衡。  
它涉及生产端、Broker 端和消费端多个环节。

## 2. 为什么重要

Kafka 往往承载高峰流量。  
一旦性能配置不合理，就容易出现积压、延迟飙升或资源耗尽。

## 3. 核心内容

- 批量发送
- 压缩
- `linger.ms`
- `batch.size`
- 分区数量设计
- 消费并发与积压治理

## 4. 学习重点

- 理解吞吐和延迟常常要做平衡
- 理解性能问题不只在 Producer 配置
- 理解积压治理离不开消费模型和下游能力

## 5. 常见问题

- 一味追求更大 batch
- 消费端处理慢却只调 Broker 参数
- 分区数量设计脱离业务规模

## 6. 练习建议

- 对比不同批量配置的效果
- 设计一个积压排查流程
- 总结 Kafka 调优的常见指标

## 7. 自测问题

- 为什么 Kafka 调优要同时看吞吐和延迟
- 消息积压通常会从哪些环节产生
- 分区数设计为什么会影响整体性能模型
