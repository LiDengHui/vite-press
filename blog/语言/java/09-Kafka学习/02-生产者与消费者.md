# 生产者与消费者

## 1. 这是什么

生产者负责发送消息，消费者负责拉取和处理消息。  
这是 Kafka 业务接入层最直接的两个角色。

## 2. 为什么重要

大多数消息链路问题，都发生在生产、消费和重试处理这几个环节。  
理解两端行为，是真正用好 Kafka 的基础。

## 3. 核心内容

- 生产发送流程
- 分区选择策略
- 消费拉取流程
- 消费组再均衡
- 提交 offset 的方式

## 4. 学习重点

- 理解发送成功不等于业务真正完成
- 理解自动提交和手动提交的差异
- 理解再均衡会影响消费稳定性

## 5. 常见问题

- 只关注发消息，不关注消费端处理语义
- 自动提交导致消息处理不一致
- 消费组变动时忽视再均衡成本

## 6. 练习建议

- 写一个简单生产消费 demo
- 对比自动提交和手动提交 offset
- 记录一次消费组再均衡过程

## 7. 自测问题

- 自动提交 offset 有什么风险
- 再均衡为什么会影响消费稳定性
- 生产端和消费端应该分别关注哪些可靠性问题
