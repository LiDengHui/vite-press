# 数据模型与节点类型

## 1. 这是什么

ZooKeeper 采用树形层级结构存储数据，节点被称为 znode。  
不同节点类型决定了数据的生命周期和使用方式。

## 2. 为什么重要

ZooKeeper 的很多上层应用场景，都是建立在节点模型基础上的。  
如果不理解节点类型，很难正确设计注册、锁和选主方案。

## 3. 核心内容

- znode
- 持久节点
- 临时节点
- 顺序节点
- 树形路径结构

## 4. 学习重点

- 理解 ZooKeeper 适合存少量元数据
- 理解临时节点和会话强绑定
- 理解顺序节点对分布式协调很重要

## 5. 常见问题

- 把 ZooKeeper 当作通用数据库使用
- 不理解临时节点消失条件
- 节点路径设计混乱
