CSS 最佳实践是编写可维护、高性能、可访问的样式代码的指导原则。本文汇总了现代 CSS 开发的核心实践，涵盖命名规范、架构设计、性能优化和可访问性等方面。

命名规范 ​

BEM 方法论 ​

css
/* Block（块） */
.card { }

/* Element（元素） */
.card__title { }
.card__image { }
.card__content { }

/* Modifier（修饰符） */
.card--featured { }
.card--large { }
.card__title--highlighted { }
html
<article class="card card--featured">
  <img class="card__image" src="photo.jpg" alt="">
  <h2 class="card__title card__title--highlighted">标题</h2>
  <div class="card__content">内容</div>
</article>

实用建议 ​

css
/* ✅ 使用语义化名称 */
.btn-primary { }
.nav-link { }
.hero-section { }

/* ❌ 避免具体值和外观描述 */
.blue-button { }    /* 颜色可能变化 */
.big-text { }       /* 大小可能变化 */
.left-sidebar { }   /* 方向可能变化（RTL） */

/* ✅ 使用动作和状态 */
.is-active { }
.is-disabled { }
.has-error { }
.is-loading { }

选择器策略 ​

特异性管理 ​

css
/* ✅ 保持低特异性 */
.btn { }
.btn-primary { }

/* ❌ 避免过度嵌套和高特异性 */
#header .nav ul li a { }  /* 特异性 1,1,3 */
body.homepage .sidebar .widget h3 { }  /* 过于具体 */

/* ✅ 使用类选择器 */
.nav-link { }  /* 特异性 0,1,0 */

避免 !important ​

css
/* ❌ 避免 */
.btn { color: red !important; }

/* ✅ 使用级联层 */
@layer base, components, utilities;

@layer utilities {
  .text-red { color: red; }
}

/* ✅ 或增加特异性 */
.btn.btn-primary { color: blue; }

选择器性能 ​

css
/* ✅ 高效 */
.header { }
.nav-item { }

/* ⚠️ 避免过度通用 */
[data-type="button"] { }  /* 属性选择器较慢 */
*:not(.exclude) { }       /* 通配符昂贵 */

/* ✅ 具体但不过度 */
.btn[data-variant="primary"] { }

聚焦状态 ​

  outline: 2px solid var(--color-primary);
  outline-offset: 2px;
}

/* ✅ 不破坏布局 */

  box-shadow: 0 0 0 2px white, 0 0 0 4px var(--color-primary);

/* ❌ 不要移除焦点 */

减少动画 ​

css

  *,
  *::before,
  *::after {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;

CSS 最佳实践的核心原则：

  1. 命名清晰：使用语义化、可预测的命名
  2. 低特异性：保持选择器简单，避免特异性战争
  3. 组件化：将 UI 拆分为独立、可复用的组件
  4. 移动优先：从最小屏幕开始，逐步增强
  5. 性能优先：关注关键渲染路径和动画性能
  6. 可访问性：确保所有用户都能使用
  7. 使用现代特性：拥抱容器查询、级联层、嵌套等新特性
  8. 代码规范：保持一致的代码风格和文档
