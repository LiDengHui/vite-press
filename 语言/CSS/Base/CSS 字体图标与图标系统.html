图标是 UI 设计的重要组成部分。CSS 可以通过字体图标（Icon Fonts）、CSS 绘制、SVG 和图片等多种方式实现图标系统。本文深入讲解各种方案的优劣势、实现方式和最佳实践。

图标实现方案对比 ​

方案优点缺点适用场景
字体图标缩放无失真、可改变颜色只单色、无法复杂效果简单线性图标
CSS 绘制无额外请求、纯 CSS复杂图标困难简单几何图形
SVG缩放无失真、可动画、多色文件稍大复杂图标、需要动画
图片支持复杂效果缩放失真、多倍图复杂效果图标

字体图标 ​

基础使用 ​

html
<!-- Font Awesome 示例 -->
<link rel="stylesheet" href="https://cdnjs.cloudflare.com/ajax/libs/font-awesome/6.4.0/css/all.min.css">

<i class="fas fa-home"></i>
<i class="fas fa-user"></i>
<i class="fas fa-cog"></i>
css
/* 自定义字体图标 */
@font-face {
  font-family: 'MyIcons';
  src: url('my-icons.woff2') format('woff2');
  font-display: swap;
}

.icon {
  font-family: 'MyIcons';
  font-style: normal;
  font-weight: normal;
  font-variant: normal;
  text-transform: none;
  line-height: 1;
  -webkit-font-smoothing: antialiased;
  -moz-osx-font-smoothing: grayscale;
}

.icon-home::before { content: '\e900'; }
.icon-user::before { content: '\e901'; }
.icon-settings::before { content: '\e902'; }

字体图标的问题 ​

css
/* 无法正确读屏的问题 */
.icon::before {
  content: '\e900';
  /* 读屏软件可能会读取为 "e900" */
}

/* 解决方案 */
.icon[aria-label]::before {
  content: attr(aria-label);
  position: absolute;
  width: 1px;
  height: 1px;
  padding: 0;
  margin: -1px;
  overflow: hidden;
  clip: rect(0, 0, 0, 0);
  white-space: nowrap;
  border: 0;
}

图标系统设计 ​

设计原则 ​

  1. 一致性：所有图标保持同一视觉风格
  2. 缩放性：在不同尺寸下保持清晰
  3. 可访问性：添加适当的 ARIA 标记
  4. 性能：优先考虑缓存和复用

可访问性 ​

html
<!-- 装饰性图标 -->
<i class="icon-home" aria-hidden="true"></i>

<!-- 有功能的图标 -->
<button aria-label="关闭">
  <i class="icon-close" aria-hidden="true"></i>
</button>

<!-- 带文本的图标 -->
<a href="/profile">
  <i class="icon-user" aria-hidden="true"></i>
  用户中心
</a>

技巧一：纯 CSS 复选框 ​

css

  appearance: none;
  width: 20px;
  height: 20px;

  position: relative;

  background: #3498db;

  top: 1px;
  width: 6px;
  height: 12px;
  border: solid white;
  border-width: 0 2px 2px 0;

总结 ​

CSS 图标系统的选择需根据具体场景决定：

  1. 字体图标：简单线性图标，需要可缩放和改色
  2. CSS 绘制：简单几何图形，无额外请求
  3. SVG：复杂图标、需要动画或多色
  4. 混合方案：根据需求组合使用
